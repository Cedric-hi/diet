# Product Contract: Diet Tracker

## Core Features

1. **Google 로그인** - Supabase Auth를 통한 Google OAuth 인증
2. **식사 기록** - 날짜 + 식사시간(아침/점심/저녁/간식) + 사진 업로드
3. **AI 식단 분석** - Gemini 3.1 Flash-Lite가 사진을 분석하여 음식명, 분량, 칼로리 산출
4. **이미지 최적화** - 업로드 사진을 webp 변환, 최대 너비 800px로 리사이즈 후 Supabase Storage 저장
5. **대시보드** - 날짜별 식사 목록 조회, 일일 칼로리 요약

## Primary User Flows

### Flow 1: 로그인
1. 랜딩 페이지 접속
2. "Google로 로그인" 버튼 클릭
3. Google OAuth 인증 완료
4. 대시보드로 리다이렉트

### Flow 2: 식사 기록 등록
1. 대시보드에서 "식사 기록" 버튼 클릭
2. 날짜 선택 (기본값: 오늘)
3. 식사 시간 선택 (아침/점심/저녁/간식)
4. 식사 사진 촬영 또는 갤러리에서 선택
5. 업로드 → 서버에서 webp 변환(800px) → Storage 저장
6. Gemini API가 사진 분석 → 음식명, 칼로리 결과 반환
7. 분석 결과 확인 후 저장
8. 대시보드로 돌아감 (새 기록 반영)

### Flow 3: 식단 조회
1. 대시보드에서 오늘 날짜의 식사 목록 확인
2. 날짜 탐색 (이전/다음일) 으로 다른 날짜 조회
3. 각 식사 카드에서 사진, 음식명, 칼로리 확인
4. 일일 총 칼로리 요약 확인

## Acceptance Criteria

### 로그인
- [ ] Google OAuth로 로그인/로그아웃 가능
- [ ] 미인증 유저는 /dashboard, /meals 접근 불가 (로그인 페이지로 리다이렉트)
- [ ] 최초 로그인 시 public.users에 자동 등록 (DB trigger)

### 식사 기록
- [ ] 날짜 선택기에서 날짜 변경 가능, 기본값은 오늘
- [ ] 4개 식사시간 중 하나 선택 필수
- [ ] 사진 업로드 시 webp로 변환, 최대 너비 800px
- [ ] Supabase Storage에 meal-images/{user_id}/{uuid}.webp 경로로 저장
- [ ] Gemini가 음식 사진을 분석하여 JSON 형태로 결과 반환 (foods[], total_calories)
- [ ] 분석 결과를 meals 테이블에 저장
- [ ] 저장 후 대시보드에 즉시 반영 (revalidatePath)

### 대시보드
- [ ] 선택된 날짜의 식사 목록 표시 (사진 썸네일 + 음식명 + 칼로리)
- [ ] 일일 총 칼로리 합계 표시
- [ ] 날짜 이전/다음 탐색 가능
- [ ] 본인 데이터만 조회 가능 (RLS)

## Core E2E Scenarios

1. **비인증 접근 차단**: 로그아웃 상태에서 /dashboard 접근 → 로그인 페이지로 리다이렉트
2. **식사 기록 전체 플로우**: 로그인 → 식사 기록 → 사진 업로드 → Gemini 분석 → 저장 → 대시보드에서 확인
3. **날짜별 조회**: 여러 날짜에 식사 기록 → 날짜 탐색으로 각 날짜의 기록만 표시
4. **데이터 격리**: 다른 유저의 식사 기록은 조회 불가
